### Description

Fixes the session automatic re-login loop and Quote API casing issue (`status` vs `success` and `errorcode` vs `errorCode`) which was causing `LTP fetch failed` errors. Additionally, adds a robust, cron-compatible initialization and maintenance wrapper script `initialize.sh`.

### Changes

- Upgraded the Axios response interceptor in `src/helpers/api.ts` to support both `status`/`success` and `errorcode`/`errorCode` casings returned by Angel One SmartAPI endpoints.
- Updated the Axios request interceptor in `src/helpers/api.ts` to omit the `Authorization` header when making the login request, preventing recursion or invalid credentials rejection.
- Created `initialize.sh` script to set up environment paths (NVM, node, pnpm) for non-interactive shells and invoke the daily maintenance script.

### Verification

- Successfully ran simulated session expiry tests via a diagnostic script verifying that automatic re-login succeeds and retries the Quote request.
- Executed `npm run verify` check (formatting check, typescript compilation, linter check, and Jest unit tests) with all tests passing cleanly.
